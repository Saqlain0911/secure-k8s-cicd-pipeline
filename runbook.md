## 🚀 Quick Start Guide

This guide walks you through provisioning AWS OIDC integration, executing the DevSecOps pipeline, and deploying a highly available application architecture to a local Kubernetes cluster.

---

## 1️⃣ Prerequisites

Ensure the following tools and configurations are available:

* AWS CLI configured with administrator credentials
* Terraform installed locally
* Docker installed locally
* Minikube installed locally
* `kubectl` configured and accessible
* A GitHub repository hosting this project

---

## 2️⃣ Provision AWS Infrastructure (Terraform)

First, create the Amazon ECR repository and establish the OIDC trust relationship between AWS and GitHub.

```bash
cd terraform
terraform init
terraform apply -auto-approve
```

> **Note:** Record the `github_actions_role_arn` output generated by Terraform. This ARN will be required when configuring the CI/CD pipeline.

---

## 3️⃣ Execute the DevSecOps Pipeline

1. Open `.github/workflows/ci-cd.yml`.
2. Locate the **Configure AWS Credentials via OIDC** step.
3. Replace the `role-to-assume` value with the IAM Role ARN generated by Terraform.
4. Commit and push the changes to the `main` branch.

```bash
git add .
git commit -m "chore: configure AWS OIDC role ARN"
git push origin main
```

✅ Navigate to the **GitHub Actions** tab in your repository. The workflow will:

* Lint the Dockerfile using Hadolint
* Build the container image
* Perform a Trivy vulnerability scan
* Authenticate to AWS using OIDC
* Push the verified image to Amazon ECR

---

## 4️⃣ Deploy the Highly Available Kubernetes Architecture

Start the local Minikube cluster and load the application image into Minikube's Docker daemon.

```bash
minikube start
minikube image load secure-k8s-app:latest
```

Deploy all Kubernetes resources:

```bash
kubectl apply -f k8s/
```

Monitor pod creation and wait until all resources reach the `Running` state:

```bash
kubectl get pods -w
```

---

## 5️⃣ Verify Resilience & Chaos Testing

Validate the NodePort service and confirm that both the health and database connectivity endpoints are functioning correctly.

```bash
# Verify readiness and liveness probes
curl http://$(minikube ip):30000/health

# Verify ConfigMap and Secret injection through database connectivity
curl http://$(minikube ip):30000/db-status
```

Simulate a database failure:

```bash
kubectl delete pod postgres-0
kubectl get pods -w
```

✅ Kubernetes automatically detects the failure, recreates the PostgreSQL pod through the StatefulSet controller, and reattaches the Persistent Volume, preserving data without manual intervention.

---

## 6️⃣ Cleanup (Cost & Resource Control)

Remove all Kubernetes resources and stop the local cluster:

```bash
kubectl delete -f k8s/
minikube stop
```

Destroy the AWS infrastructure to prevent unnecessary cloud costs:

```bash
cd terraform
terraform destroy -auto-approve
```
